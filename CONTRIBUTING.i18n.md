## English

Thanks for your interest in contributing to Kommz Gamer Community.

### Before opening a PR
- Open an issue first for major changes.
- Keep changes focused and easy to review.
- Do not include secrets, private endpoints, or personal local configuration.

### Development guidelines
- Prefer small, isolated commits.
- Document user-visible changes in `CHANGELOG.md` when relevant.
- Keep Windows desktop behavior in mind when changing startup, audio, or webview logic.

### Security / privacy
- Never commit `.env`, `settings.json`, logs, or personal audio assets.
- If you discover a vulnerability, use the process described in `SECURITY.md`.

## Francais

Merci pour votre interet pour contribuer a Kommz Gamer Community.

### Avant d'ouvrir une PR
- Ouvrez d'abord une issue pour les changements importants.
- Gardez les changements cibles et faciles a relire.
- N'incluez pas de secrets, d'endpoints prives, ni de configuration locale personnelle.

### Bonnes pratiques de developpement
- Privilegiez des commits petits et isoles.
- Documentez les changements visibles utilisateur dans `CHANGELOG.md` quand c'est pertinent.
- Gardez en tete le comportement desktop Windows pour les modifications startup, audio ou webview.

### Securite / confidentialite
- Ne jamais commit `.env`, `settings.json`, logs ou assets audio personnels.
- Si vous decouvrez une vulnerabilite, suivez le process de `SECURITY.md`.
