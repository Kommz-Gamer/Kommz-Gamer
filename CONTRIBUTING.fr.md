# Contribution

[English](CONTRIBUTING.md) | [Français](CONTRIBUTING.fr.md)

Merci pour votre intérêt pour contribuer à Kommz Gamer Community.

## Avant d’ouvrir une PR
- Ouvrez d’abord une issue pour les changements importants.
- Gardez les changements ciblés et faciles à relire.
- N’incluez pas de secrets, d’endpoints privés, ni de configuration locale personnelle.

## Bonnes pratiques de développement
- Privilégiez des commits petits et isolés.
- Documentez les changements visibles utilisateur dans `CHANGELOG.md` quand c’est pertinent.
- Gardez en tête le comportement desktop Windows pour les modifications startup, audio ou webview.

## Sécurité / confidentialité
- Ne jamais commit `.env`, `settings.json`, logs ou assets audio personnels.
- Si vous découvrez une vulnérabilité, suivez le process décrit dans `SECURITY.md`.
