# Security Policy (EN + FR)

[Go to English](#english) | [Aller au Français](#francais)

---

## English

If you discover a security issue, please do not publish it immediately in a public issue.

Please report it privately with:
- a clear description of the issue
- steps to reproduce
- impact assessment if known
- affected version(s)

Typical sensitive areas:
- API keys and secrets
- license logic
- hosted endpoints
- remote control features
- local webview / embedded server startup

---

## Francais

Si vous decouvrez un probleme de securite, merci de ne pas le publier immediatement dans une issue publique.

Merci de le signaler en prive avec :
- une description claire du probleme
- les etapes de reproduction
- une estimation de l'impact si connue
- la/les version(s) concernee(s)

Zones generalement sensibles :
- cles API et secrets
- logique de licence
- endpoints heberges
- fonctions de controle distant
- demarrage webview local / serveur embarque
