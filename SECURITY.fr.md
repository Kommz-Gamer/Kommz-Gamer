# Politique de sécurité

[English](SECURITY.md) | [Français](SECURITY.fr.md)

Si vous découvrez un problème de sécurité, merci de ne pas le publier immédiatement dans une issue publique.

Merci de le signaler en privé avec :
- une description claire du problème
- les étapes de reproduction
- une estimation de l’impact si connue
- la/les version(s) concernée(s)

Zones généralement sensibles :
- clés API et secrets
- logique de licence
- endpoints hébergés
- fonctions de contrôle distant
- démarrage webview local / serveur embarqué
