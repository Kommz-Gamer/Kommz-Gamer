# Recommended Repository Structure

This structure keeps the project clear and easy to maintain while preserving existing folders.

## Core folders
- `src/`: application code (UI, logic, integrations)
- `config/`: default and sample configuration files
- `scripts/`: automation scripts (build, release, maintenance)
- `docs/`: project documentation
- `.github/`: GitHub templates and CI/CD helpers

## Current release template
- `.github/release-template.en.md`
- `.github/release-template.fr.md`

## Suggested docs to keep in `docs/`
- setup guide
- troubleshooting
- architecture overview
- release process notes
